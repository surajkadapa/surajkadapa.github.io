SR Latch

Let's unlock the secrets of computer memory! In the next series of posts, we will explore how memory is stored in a computer and learn about the basic memory elements. By the end of the series, we will have built one register that can store bits, similar to how a computer stores data.

What is a register?

Registers are small super-fast memory storage locations made primarily using flip-flops, which are basic electronic circuits storing a single bit (0 or 1). They can hold multiple bits of data when combined.

To understand flip-flops we must first understand what latches are and let’s get started with the most basic latch.

Even within the circuit, we can observe that once B is set to 1, the output remains 1. This demonstrates the basics of memory building, where the output is retained without any input.

However, this circuit has a major issue: we cannot change the output once it is set to 1. To address this problem, we use the SR latch.

Understanding the SR Latch

The "S" and "R" in an SR latch stand for "Set" and "Reset" respectively. To understand how an SR latch works, let's examine its schematic and truth table.

The schematic above shows the SR latch, which is a circuit made up of two NOR gates. The S input (SET) is connected to the output of the other NOR gate, while the R input (RESET) is connected to the other. Q is the output of the circuit and Q’ is the complement of the output. Let’s see the truth table below to understand it even better.

Before we get to the truth table of the SR latch, let’s examine the truth table of the NOR gate.

An interesting observation about the NOR gate is that when either or both of its inputs are 1, the output is always 0. This observation will be helpful as we examine the truth table of the SR latch.

Here, S and R represent the Set and Reset inputs, respectively. When S=0 and R=1, the output is set to 0. This is because when we give 1 to the first NOR gate, the output Q becomes 0. Since Q is fed back into the second NOR gate, we get Q’ as 1.

Similarly, when S=1 and R=0, the output Q is set to 1. When both S and R are 1, the circuit enters an invalid state, and its behavior is undefined.

The table's first row describes the circuit's behavior when S and R are both 0. In this scenario, Q keeps its previous state, and Q’ is its complement. This ability to retain the previous output when Set and Reset are both 0 is what gives this circuit its name - a latch.

There is still one big problem in the SR latch, there is one metastable state when our Set and Reset are 1. This inefficiency leaves us yearning for a more optimal solution, a state where no potential is squandered. To address this, digital electronics engineers have come up with the D latch.
